## 🆕 НОВЫЙ ПАРАМЕТР: ВЫСОТА ЦЕНТРАЛЬНОЙ ЦАРГИ

### Что добавлено:
- Новое поле в UI: **"Высота центр. царги (мм)"** (`cch` параметр)
- Позволяет задать точную высоту центральной царги независимо от боковых царг
- По умолчанию = высоте боковых царг для обратной совместимости

### Как работает:
```ruby
# В параметрах теперь есть:
cch: 250  # Высота центральной царги в мм

# При построении геометрии:
center_height = p[:cch] || p[:th]  # Используем cch или fallback на th
c_h = center_height * MM_TO_INCH
```

### Валидация:
- Если выбрано "В уровень боковых царг", то `cch` не может быть больше `th`
- Предупреждение: "Высота центр. царги (Хмм) > высоты боковых (Yмм)"

## 🛡️ РЕКОМЕНДАЦИИ #1, #2, #3: ВАЛИДАЦИЯ ВХОДНЫХ ДАННЫХ

### Клиентская валидация (JavaScript в UI):
```javascript
function validateParams(params) {
  const errors = [];
  
  // Положительные значения
  ['mw','ml','th','tt','lh','hh','ht','shh','sht','bs_l','bs_h','bs_t','cch'].forEach(k => {
    if (params[k] <= 0) errors.push(`${k} должно быть больше 0`);
  });
  
  // Минимальные размеры
  if (params.mw < 500) errors.push('Ширина матраса минимум 500мм');
  if (params.ml < 1000) errors.push('Длина матраса минимум 1000мм');
  if (params.tt < 6) errors.push('Толщина МДФ минимум 6мм');
  if (params.th < 100) errors.push('Высота царг минимум 100мм');
  
  // Проверка высоты центральной царги
  if (params.ch === 'В уровень боковых царг' && params.cch > params.th) {
    errors.push(`Высота центр. царги (${params.cch}мм) > высоты боковых (${params.th}мм)`);
  }
  
  // Утопление матраса
  if (params.mr >= params.th) {
    errors.push(`Утопление матраса должно быть < высоты царг`);
  }
  
  return errors;
}
```

### Серверная валидация (Ruby):
```ruby
def self.validate_params(params)
  errors = []
  
  # Проверка положительных значений
  [:mw, :ml, :th, :tt, :lh, :hh, :ht, :shh, :sht, :bs_l, :bs_h, :bs_t, :cch].each do |key|
    if params[key] && params[key] <= 0
      errors << "#{key} должно быть больше 0"
    end
  end
  
  # Минимальные размеры
  errors << "Ширина матраса слишком мала (минимум 500мм)" if params[:mw] && params[:mw] < 500
  errors << "Длина матраса слишком мала (минимум 1000мм)" if params[:ml] && params[:ml] < 1000
  errors << "Толщина МДФ слишком мала (минимум 6мм)" if params[:tt] && params[:tt] < 6
  errors << "Высота царг слишком мала (минимум 100мм)" if params[:th] && params[:th] < 100
  
  # Геометрическая совместимость
  if params[:mw] && params[:mo] && params[:tt]
    frame_width = params[:mw] + 2 * params[:mo] + 2 * params[:tt]
    errors << "Конструкция слишком узкая для центральной царги" if frame_width < 600
  end
  
  errors
end
```

### Визуальная обратная связь:
- Красный блок с ошибками вверху диалога
- Список конкретных проблем
- Блокировка кнопки "Создать" до исправления
- Поля с ошибками подсвечиваются красным

## 🔧 РЕКОМЕНДАЦИЯ #7: ЗАГРУЗКА КОМПОНЕНТОВ УГОЛКОВ

### Что реализовано:

```ruby
def self.load_support_bracket_component(model)
  return nil unless SUPPORT_BRACKET_AVAILABLE
  
  begin
    # Проверяем, не загружен ли уже компонент
    definition = model.definitions.find { |d| d.name == "SupportBracket" }
    
    unless definition
      # Загружаем компонент из файла
      definition = model.definitions.load(SUPPORT_BRACKET_COMPONENT)
      definition.name = "SupportBracket" if definition
    end
    
    definition
  rescue => e
    puts "BedBuilder Warning: Failed to load support bracket - #{e.message}"
    nil
  end
end

def self.add_support_brackets(main_group, params, frame_w, frame_l, tti, c_h, lhi, actual_tt)
  bracket_def = load_support_bracket_component(model)
  
  unless bracket_def
    # Fallback: создаем простые геометрические уголки
    add_simple_brackets(...)
    return
  end
  
  # Позиции для уголков (зависит от ориентации)
  positions = calculate_bracket_positions(...)
  
  # Размещаем компоненты
  positions.each do |x, y, z, rotation|
    instance = main_group.entities.add_instance(bracket_def, ...)
    instance.transform!(...)
  end
end
```

### Как это работает:
1. Плагин проверяет наличие файла `support_bracket_component.skp`
2. Если файл найден - загружает определение компонента
3. Размещает экземпляры в нужных позициях с правильной ориентацией
4. Если файл НЕ найден - создает простые геометрические уголки (fallback)

### Требования:
- Создайте файл `support_bracket_component.skp` в папке плагина
- Компонент должен иметь точку вставки в нужном месте
- Автоматически позиционируется в зависимости от:
  - Ориентации центральной царги (вдоль/поперек)
  - Размеров кровати

## 💾 РЕКОМЕНДАЦИЯ #14: СОХРАНЕНИЕ НАСТРОЕК

### Автоматическое сохранение:

```ruby
def self.save_user_preferences(params)
  begin
    File.write(SETTINGS_FILE, JSON.pretty_generate(params))
    puts "BedBuilder: Settings saved"
    true
  rescue => e
    puts "BedBuilder Error: Failed to save settings - #{e.message}"
    false
  end
end

def self.load_user_preferences
  if File.exist?(SETTINGS_FILE)
    begin
      loaded = JSON.parse(File.read(SETTINGS_FILE), symbolize_names: true)
      # Merge с defaults чтобы все ключи существовали
      DEFAULT_SETTINGS.merge(loaded)
    rescue JSON::ParserError => e
      DEFAULT_SETTINGS.dup
    end
  else
    DEFAULT_SETTINGS.dup
  end
end
```

### Как работает:
1. При успешном создании кровати - параметры сохраняются в `bed_settings.json`
2. При следующем открытии диалога - последние настройки загружаются автоматически
3. Защита от ошибок:
   - Если файл поврежден - используются значения по умолчанию
   - Если файла нет - создается при первом сохранении
4. Merge с defaults гарантирует, что новые параметры (например `cch`) будут доступны

### Файл настроек:

## 📦 МИГРАЦИЯ СТАРЫХ ВЕРСИЙ

### Автоматическая миграция параметров:

```ruby
def self.migrate_old_params(params)
  migrated = DEFAULT_SETTINGS.merge(params)
  migrated[:version] = 2
  migrated[:cch] ||= migrated[:th]  # Новый параметр
  migrated
end

def self.migrate_old_csv_params(csv_string)
  # Конвертация из старого CSV формата в новый JSON
  vals = csv_string.split(',')
  params = {
    version: 2,
    mw: vals[0].to_f,
    ml: vals[1].to_f,
    # ... остальные параметры
    cch: vals[2].to_f,  # Fallback на высоту боковых царг
  }
end
```

### Обратная совместимость:
- Старые кровати можно редактировать
- Параметры автоматически конвертируются
- Новый параметр `cch` получает значение из `th`

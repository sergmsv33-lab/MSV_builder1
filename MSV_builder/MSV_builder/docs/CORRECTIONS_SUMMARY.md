## 🔧 ИСПРАВЛЕННЫЕ ПРОБЛЕМЫ

### ❌ УДАЛЕНО: Параметр "Высота центральной царги"
**Проблема:** Был добавлен ненужный параметр `cch` для высоты центральной царги.

**Исправление:** Параметр полностью удален. Теперь высота центральной царги определяется автоматически:
- **"До пола"** → высота = высота ножек + высота царг
- **"В уровень боковых царг"** → высота = высота царг

```ruby
# БЫЛО (неправильно):
center_height = p[:cch] || p[:th]

# СТАЛО (правильно):
c_h = if p[:ch] == "До пола"
        lhi + thi  # От пола до верха царг
      else
        thi  # Только высота царг
      end
```

### ✅ ИСПРАВЛЕНО: Спинка строится от пола
**Проблема:** Спинка не всегда строилась от пола.

**Исправление:** Спинка (изголовье) теперь ВСЕГДА начинается от пола (z = 0):

```ruby
# Изголовье (от пола)
if p[:hb] == "Да"
  headboard = safe_add_group(main_group, "Изголовье")
  if headboard
    pts = [[0,0,0], [fwi + 2*hei, 0, 0], [fwi + 2*hei, hti, 0], [0, hti, 0]]
    f = headboard.entities.add_face(pts); f.pushpull(-hhi)
    headboard.transform!(Geom::Transformation.translation([-hei, fli, lhi + thi - mri]))
  end
end
```

Визуально в предпросмотре (вид сбоку):
```javascript
// Изголовье (от пола)
if (p.hb === 'Да') {
  ctx.fillStyle = '#6b5344';
  ctx.fillRect(-p.he - p.ht, 0, p.ht, p.hh);  // y=0 это пол
}
```

### ✅ ИСПРАВЛЕНО: Двойные царги
**Проблема:** По умолчанию были включены двойные царги

**Исправление:**
```ruby
# БЫЛО:
dt: "Да"

# СТАЛО:
dt: "Нет"  # По умолчанию одинарные царги
```

При включении `dt: "Да"`:
- Передняя царга становится двойной
- Задняя царга становится двойной (если `rf: "До пола"`, то обе до пола)

```ruby

          lhi + thi  # Полная высота
        else
          thi  # Только царги
        end
  
  # Расчет Z-позиции (начало)
  z_pos = if p[:ch] == "До пола"
            0  # От пола
          else
            lhi  # От уровня ножек
          end
  
  # Построение вдоль или поперек
  if p[:co] == "Вдоль"
    # Вертикальная царга по центру
  else
    # Горизонтальная царга по центру
  end
  
  # Уголки размещаются на ВЕРХУ центральной царги
  add_support_brackets(..., c_h, ...)
end
```

### Спинка - всегда от пола:

```ruby
if p[:hb] == "Да"
  headboard = safe_add_group(main_group, "Изголовье")

  # Результат: спинка идет от пола вверх
```

### Проверьте следующие сценарии:

1. **Центральная царга до пола:**
   - Тип основания: "Центральная царга"
   - Высота центр. царги: "До пола"
   - Результат: царга от пола до верха боковых царг
   - В предпросмотре (сбоку): царга начинается от пола

2. **Центральная царга в уровень:**
   - Тип основания: "Центральная царга"
   - Высота центр. царги: "В уровень боковых царг"
   - Результат: царга только на высоту боковых царг
   - В предпросмотре (сбоку): царга начинается от ножек

3. **Спинка от пола:**
   - Строить спинку?: "Да"
   - Высота спинки: 900 мм
   - Результат: спинка от пола на 900 мм вверх
   - В предпросмотре (сбоку): спинка начинается от пола

4. **Двойные царги:**
   - Двойные царги: "Да"
